# AIGuide iOS 产品路线图

## 北极星

把 AIGuide 做成一个“现场感知型 AI 讲解员”：用户到达一个地点、看到一个展品或搜索一个景点时，App 能知道上下文，生成可靠、好听、可追问、可继续规划的讲解。

## 阶段 1：可信核心闭环

状态：已完成 MVP

- 定位后自动识别当前或附近 POI。
- 显示匹配状态、路线进度、置信度和来源。
- 支持用户手动纠错和拍照确认。
- 后端慢或不可用时保留本地 fallback。

## 阶段 2：导览体验打磨

状态：已完成第一轮

- 底部抽屉支持 collapsed / half / expanded。
- 避免抽屉和底部 Tab、地图控件互相遮挡。
- 主视觉从固定故宫贴图改成更通用的地图和内容层。
- 支持播放、暂停、快问、来源、纠错、讲解风格切换。

## 阶段 3：识景与追问

状态：已完成 MVP，待生产化

- 拍照和相册识别入口可用。
- 接入本地 Vision fallback。
- 预留后端多模态模型识别链路。
- 识别结果可反向确认当前导览点位。
- 支持围绕识别结果继续追问。

## 阶段 4：行程生成

状态：已完成 MVP，待增强

- 搜索景点生成行程。
- 热门重名景点有初步 disambiguation。
- 行程详情、站点、播放条和每日回顾可用。
- 对旧生成数据增加显示层本地化。

下一步：

- 用真实地点知识库和 LLM 共同生成更高质量的站点讲解。
- 支持用户选择游玩时长、兴趣、人群、节奏。
- 支持同城多景点串联和室内展馆路线。

## 阶段 5：全球化与多语言

状态：第一轮完成

- 已接入 `Localizable.xcstrings`。
- 支持简体中文、繁体中文、英文、日文、韩文。
- App 内语言设置会注入 SwiftUI 环境。

下一步：

- 把 LLM prompt、后端请求和内容缓存全面带上语言上下文。
- 增加伪本地化和长文本 UI QA。
- 对景点名采用“本地名 + 用户语言说明”的策略。

## 阶段 6：生产化

状态：未完成

- 用户登录与收藏同步。
- 权限、隐私、数据删除与合规说明。
- 后端内容审核、来源管理、LLM 成本控制。
- 真实离线包下载、增量更新、缓存淘汰。
- TestFlight 内测、崩溃日志、埋点分析。

## 暂不优先

- 机票/酒店预订。
- 泛餐厅点评和交易闭环。
- 大而全地图导航。

这些方向会让产品变成携程/大众点评/地图 App 的弱替代。短期差异化应集中在“现场讲解、看见就问、路线成章、可信来源”。
