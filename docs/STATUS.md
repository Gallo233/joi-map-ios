# 完成与未完成清单

## 已完成

### 基础工程

- SwiftUI iOS 工程可构建。
- iOS 17+ 目标。
- `Info.plist` 权限覆盖定位、相机、麦克风、语音识别、相册。
- 主 Tab：Guide、Scan、Trips、Settings。

### 导览

- MapKit 地图主界面。
- 根据定位和附近 POI 更新当前讲解对象。
- 附近文化路线和当前站点进度。
- 可拖动抽屉和播放控制。
- 讲解来源、纠错、风格切换、追问入口。
- 本地缓存和弱网 fallback。

### 识景

- 拍照识别和从相册选择入口。
- 本地 Vision fallback。
- 识别成功后可确认 POI 并回填导览。
- 识别结果页有来源、置信度、候选和追问输入。

### 行程

- 搜索景点入口。
- 生成行程的 UI 和数据结构。
- 推荐路线、我的行程、今日回顾。
- 行程详情页、时间线、站点卡片、底部播放器。
- 对热门重名地点有初步过滤，避免住宅区/商业楼误当景点。

### 设置

- 自动播放、语音、默认风格、地图样式。
- 语音设置支持本机试听，选择后会持久化。
- 外观模式：跟随系统、浅色、深色。
- App 内语言选择。
- 后端诊断、离线内容管理、本机数据管理、通知、触感、缓存等基础项。

### 本地化

- `Localizable.xcstrings` 已补齐五种语言。
- 当前检查结果：
  - `missing_localizations = 0`
  - `referenced_missing = 0`
- 已验证 Guide、Scan、Trips、Settings 和 Themed Routes 主路径英文显示正常。

## 未完成

### 后端与模型

- 客户端 `APIClient` 已统一 API root/server root 配置、语言上下文注入、Accept-Language 请求头、2xx 状态处理、瞬时错误重试和后端错误信息解析。
- 需要稳定的后端部署地址。
- 需要把 MiMo/其他多模态模型通过后端代理接入，避免客户端持有真实 key。
- 需要把后端错误码和限流策略与产品 UI 做端到端对齐。
- 需要内容安全和来源校验。

### 内容质量

- 当前仍混合 mock、MapKit、LLM 生成和本地 fallback。
- 需要真实景点知识库。
- 需要为不同国家、景点类型建立 prompt 模板。
- 需要更强的重名景点 disambiguation。

### QA

- 小屏/Pro/Max 多尺寸截图回归。
- 深色/浅色/跟随系统回归。
- 位置拒绝、低精度、无网络、慢网络测试。
- 相机不可用、相册权限拒绝、语音权限拒绝测试。
- 真机性能、发热、耗电测试。

### 产品化

- 登录、收藏、历史同步。
- 离线包下载和管理。
- 支付/订阅策略。
- 账号云端数据删除流程和合规复核。
- TestFlight 打包和分发。

## 已知限制

- Apple Maps 底图标签语言不完全由 App 控制。
- 模拟器定位和 VPN 会影响地图中心和附近候选。
- 历史生成行程已加显示层本地化，但用户自定义输入会保持原文。
- 部分旧实验页仍保留在代码中，已处理主要入口，后续可以继续收敛或删除。
